# 🤖 Redrob AI Advanced Candidate Ranking Engine

Production-grade Intelligent Candidate Discovery pipeline for the **India Runs Data & AI Challenge**.  
Ranks 100,000 candidates against a Senior AI Engineer JD in **< 5 minutes on CPU**, outputs top-100 to a validated CSV.

## 🚀 Quick Start

### 1. Install dependencies
```bash
pip install -r requirements.txt
```

### 2. Run offline embedding precomputation (one-time, ~10 min)
```bash
python backend/embedding_cache.py --data-dir data/ --batch-size 256
```
Outputs: `data/embeddings_matrix.npy` and `data/candidate_ids.npy`

### 3. Run the ranker (< 5 min)
```bash
python backend/ranker.py --data-dir data/ --jd data/job_description.docx --out submission.csv
```

### 4. Validate submission
```bash
python validate_submission.py submission.csv
```

### 5. Launch Streamlit UI
```bash
streamlit run frontend/app.py
```

### 6. Smoke test
```bash
python smoke_test.py
```

## 🏗️ Architecture: Dual-Mode Lifecycle

### Offline Phase (one-time)
- Streams 100k JSONL records in batches of 256
- Concatenates `profile.summary` + all `career_history.description` blocks per candidate
- Encodes with `all-MiniLM-L6-v2` on CPU (L2-normalised for fast dot-product)
- Saves `(N, 384)` float32 matrix + companion ID index

### Online Phase (< 5 min)
1. **Stage 1** — NumPy dot-product selects top-2,000 semantically similar candidates  
2. **Stage 2** — Vectorized filters apply hard masks + soft penalties:
   - Hard zero: excluded titles (HR, Marketing, Accounting, Civil/Mechanical Eng, etc.)
   - Hard zero: honeypot detection (low completeness + inflated endorsements)
   - Penalty: pure outsourcing-company history → `0.4×`
   - Penalty: job-hopper (avg tenure < 18 months) → `0.6×`
   - Boost: open-to-work flag → `1.1×`
   - Boost: verified email + phone → `1.05×`
   - Boost: AI/ML title keywords → `1.2×`
3. **Stage 3** — Redrob Velocity Scalar (V_s) from 23 behavioral signals
4. **Composite score**: `0.45 * semantic + 0.30 * V_s + 0.15 * filter + 0.10 * education`
5. Sort descending; alphabetical tie-break; top 100 → `submission.csv`

---

## 📊 Scoring Formula

$$V_s = \text{recruiter\_response\_rate} \times \left(1 - \frac{\text{notice\_period\_days}}{180}\right) \times \log(\text{github\_activity\_score} + 2)$$

Combined with:
- Activity decay: `exp(-days_inactive / 90)`
- Work-mode boost for hybrid/flexible in Indian tech hubs
- Interview completion rate × offer acceptance rate reliability
- Platform engagement: log-scaled views, saves, searches

## 🛡️ Honeypot & Trap Detection

| Trap Type | Detection Logic | Effect |
|-----------|----------------|--------|
| Excluded Role | HR/Marketing/QA/Civil/Mech title | Score → 0 |
| Honeypot Profile | completeness < 50 AND avg_endorsements > 30 | Score → 0 |
| Extreme Incompleteness | completeness < 25 | Score → 0 |
| Pure Outsourcing | All career at TCS/Infosys/Wipro/etc., no product co. | Score × 0.4 |
| Job-Hopper | Avg tenure < 18 months | Score × 0.6 |
